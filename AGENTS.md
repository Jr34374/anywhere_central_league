# WebARプロジェクト「AR_demo」AIエージェント用指示書

## 1. プロジェクト概要
- **目的:** スマートフォンのGPSとコンパスを利用し、指定した現実の座標（緯度・経度）の方向にARオブジェクトを表示する、ロケーションベースのWebARアプリを開発する。
- **ターゲット:** スマホの標準Webブラウザ（Android Chrome等）。ネイティブアプリのインストールは不要。

## 2. 技術スタック
- HTML / JavaScript
- A-Frame (3D描画用のWebフレームワーク)
- AR.js (Location-basedコンポーネントを利用したWebARエンジン)
- 必ずしもシングルファイル構成である必要はないですが、できるだけシンプルな構成で実装すること

## 3. 実装すべき必須要件
1. **カメラ連動:** デバイスのカメラ映像を背景としてリアルタイムに描画する。
2. **位置情報に基づくAR配置:** セ・リーグ全本拠地球場の以下の座標に、認識しやすい3Dオブジェクト（`<a-box>`やテキストなど）を配置する。
   - バンテリンドーム ナゴヤ: lat: 35.1859, lon: 136.9474
   - 明治神宮野球場: lat: 35.6744, lon: 139.7170
   - 阪神甲子園球場: lat: 34.7213, lon: 135.3616
   - MAZDA Zoom-Zoom スタジアム広島: lat: 34.3919, lon: 132.4847
   - 横浜スタジアム: lat: 35.4433, lon: 139.6401
   - 東京ドーム: lat: 35.7056, lon: 139.7519
3. **距離の表示:** ユーザーの現在地（GPS）から各目標地点までの距離を計算し、画面上にテキストUIとして表示する。
4. **権限リクエスト:** iOS環境等で必要となる「カメラ」「位置情報」「デバイスの向き（DeviceOrientation API）」の取得許可を求める処理（ユーザーのタップ操作による開始など）を実装する。
